Introduction

Welcome to our WhatsApp API documentation. This guide will walk you through the available endpoints and authentication process to interact with WhatsApp using our API.

General

Our API is based on instances, each equivalent to a WhatsApp account. From our APP or our API, you can create, modify, and delete instances. After creating an instance, link it to a WhatsApp Web account by scanning the provided QR code with a mobile device. Once linked, you can perform all available tasks: search for contacts, chats, view messages, download multimedia, and send text and multimedia messages.

Authentication

Base URL: https://api.chatter.com

To access our API, you need to authenticate using JSON Web Tokens (JWT). Please include your JWT in the Authorization header for every request.

     
  curl --location --request POST 'https://chatter.plus/api' \
  --header 'Authorization: Bearer YOUR-API-TOKEN-HERE' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json'

Endpoints

Create an Instance: /api/instance/create

Description: Create a WhatsApp instance.

Method: POST

Authentication: Requires JWT. (See click here)

Input Parameters (body JSON params):

  • name: ID of the instance. (Required)
  • webhook: URL where you want us to notify you with a POST request when you receive a message. (Optional)

Delete an Instance: /api/instance/delete

Description: Delete a WhatsApp instance.

Method: POST

Authentication: Requires JWT. (See click here)

Input Parameters (body JSON params):

  • instanceId: ID of the instance. (Required)

Edit an Instance: /api/instance/edit

Description: Edit a WhatsApp instance.

Method: POST

Authentication: Requires JWT. (See click here)

Input Parameters (body JSON params):

  • instanceId: ID of the instance. (Required)
  • name: ID of the instance. (Optional)
  • webhook: URL where you want us to notify you with a POST request when you receive a message. (Optional)

Get chat messages /api/instance/chat

Description: Get your conversation with an user or group.

Method: POST

Authentication: Requires JWT. (See click here)

Input Parameters (body JSON params):

  • instanceId: ID of the instance. (Required)
  • remoteJid: ID of user or group (ej phonenumber@whatsapp.net or groupid@g.us).(Required)
  • qty: Quantity of messages you want receive. (Optional)

Send Message: /api/instance/send

Description: Allows sending a message through an active WhatsApp instance.

Method: POST

Authentication: Requires JWT. (See click here)

Input Parameters (body JSON params):

  • instanceId: ID of the instance. (Required)
  • remoteJid: ID of user or group (ej phonenumber@whatsapp.net or groupid@g.us).(Required)
  • message: String you want to send. (Required)

Send Media Message: /api/instance/sendmedia

Description: Allows sending a message through an active WhatsApp instance.

Method: POST

Authentication: Requires JWT. (See click here)

Input Parameters (body JSON params):

  • instanceId: ID of the instance. (Required)
  • remoteJid: ID of user or group (ej phonenumber@whatsapp.net or groupid@g.us).(Required)
  • fileUrl: URL of your media. (Required)
  • caption: Type of your media ('document', 'image', 'audio', 'video'). (Required)
  • type: Type of your media ('document','image','audio','video'). (Required)
  • mimetype: String indicating the format of your file. (Required, e.g., image/png, video/mp4 -> Check mimetypes)
  • document: Boolean. True if you want to send as a file. (Optional, default value: false)
  • fileName: String indicating the name of your file when 'document' value is true. (Optional)

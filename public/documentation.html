Introduction

Welcome to our WhatsApp API documentation. This guide will walk you through the available endpoints and authentication process to interact with WhatsApp using our API.

Authentication

To access our API, you need to authenticate using JSON Web Tokens (JWT). Please include your JWT in the Authorization header for every request.

     curl -X GET \
    -H "Authorization: Bearer your-api-token" \
    -H "Accept: application/json" \
    -H "Content-Type: application/json" \
    https://api.yourcompany.com/endpoint

Base URL

https://api.yourcompany.com

Endpoints

Initialize an Instance: /api/instance/init

Description: Initializes a WhatsApp instance for active usage.

Method: POST

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to initialize.

Successful Response: 200 OK if the instance initializes successfully.

Error Response: Specific error code and message if initialization fails.

Delete an Instance: /api/instance/delete

Description: Permanently deletes a WhatsApp instance.

Method: DELETE

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to delete.

Successful Response: 204 No Content if the instance is deleted successfully.

Error Response: Specific error code and message if deletion fails.

Edit an Instance: /api/instance/edit

Description: Allows modification of details for an existing WhatsApp instance.

Method: PUT

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to edit.
  • Updated instance details (e.g., name, additional configurations).

Successful Response: 200 OK if the instance is edited successfully.

Error Response: Specific error code and message if editing fails.

Get Contacts of an Instance: /api/instance/contacts

Description: Retrieves the list of contacts associated with a WhatsApp instance.

Method: GET

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to fetch contacts from.

Successful Response: 200 OK and the list of contacts if successful.

Error Response: Specific error code and message if operation fails.

Send Message to an Instance: /api/instance/send

Description: Allows sending a message through an active WhatsApp instance.

Method: POST

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to send the message to.
  • Message data (content, recipient, message type).

Successful Response: 200 OK if the message is sent successfully.

Error Response: Specific error code and message if sending fails.

Get Chat History of an Instance: /api/instance/chat

Description: Retrieves the complete chat history of a WhatsApp instance.

Method: GET

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to fetch chat history from.

Successful Response: 200 OK and the complete chat history if successful.

Error Response: Specific error code and message if operation fails.

Get Message Details: /api/instance/message

Description: Retrieves specific details of a sent or received message in a WhatsApp instance.

Method: GET

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to fetch message details from.
  • messageId: Unique ID of the message to get details of.

Successful Response: 200 OK and message details if successful.

Error Response: Specific error code and message if operation fails.

Download Media of an Instance: /api/instance/media

Description: Allows downloading multimedia files (images, videos, documents) sent through a WhatsApp instance.

Method: GET

Authentication: Requires JWT.

Input Parameters:

  • instanceId: ID of the instance to download media from.
  • mediaId: Unique ID of the multimedia file to download.

Successful Response: The requested multimedia file if successful.

Error Response: Specific error code and message if operation fails.

Implementation Examples

Each operation can be implemented in various programming languages using suitable HTTP request libraries. Ensure proper handling of authentication and response parsing for each request.
